# 慢病用药小管家 - 产品方案

## 1. 产品目标

围绕慢病患者长期用药场景，设计一个网页应用，解决以下关键问题：

- 容易忘记按时服药
- 不清楚复杂药单如何执行
- 药快吃完才发现，影响连续治疗
- 漏服后不知道如何处理
- 复诊前很难整理近期用药和身体变化

产品的核心价值不是单次提醒，而是建立连续、低负担、可执行的长期用药管理机制。

## 2. 用户定位

主要用户：

- 高血压、糖尿病、高血脂等长期服药患者
- 需要子女协助管理用药的中老年患者

延伸用户：

- 家属照护者
- 复诊前需要整理服药情况的患者

## 3. 产品主线

主闭环：

问诊后建档 -> 生成用药计划 -> 每日提醒与打卡 -> 库存预警 -> 复诊/续方提醒 -> AI辅助解释

首页只解决一个核心问题：

`我今天该做什么？`

## 4. 页面结构

### 首页 Dashboard

核心模块：

- 顶部欢迎区和今日完成率
- 下一次服药提醒卡
- 今日日程时间轴
- 药品库存预警
- 复诊提醒
- 快捷入口：上传药单、漏服补救、AI问答

### 用药计划 Medication Plan

核心模块：

- 药物清单
- 剂量 / 频次 / 饭前饭后
- 时间分布
- 新增或编辑药品

### 打卡记录 Adherence

核心模块：

- 服药打卡
- 漏服标记
- 近期依从性趋势
- 症状与副作用备注

### 续方与复诊 Refill & Follow-up

核心模块：

- 药品剩余天数
- 建议续方日期
- 下次复诊时间
- 复诊摘要生成

## 5. 加分创新点

### AI 药单解析

输入处方文字或截图，自动整理为结构化药单：

- 药名
- 剂量
- 频次
- 服用时机
- 注意事项

### 漏服补救助手

用户输入原定服药时间和当前时间，系统给出保守型建议和风险提示，缓解焦虑。

### 续方风险预测

根据当前库存和服药频率，提示预计剩余天数与推荐续方时间。

## 6. 设计原则

- 简洁优先，避免信息堆叠
- 首屏聚焦今日任务
- 操作大按钮、少层级，适合中老年用户
- 提示温和，不制造紧张感
- 默认内置示例数据，保证评审打开即有内容

## 7. 视觉方向

- 主色：低饱和青绿色
- 背景：暖白和浅灰层次
- 强调色：琥珀黄用于提醒
- 风格关键词：安心、清晰、可信赖、生活化

## 8. MVP 范围

首版优先完成：

1. 首页总览
2. 用药计划页
3. 服药打卡
4. 库存预警
5. 复诊提醒
6. 漏服补救入口

## 9. 评审展示策略

- 打开即进入完整 Demo 用户视角
- 不依赖注册登录
- 所有关键卡片可点击
- 一屏即可看到核心价值
- 页面简洁但信息足够丰富，体现产品思考
